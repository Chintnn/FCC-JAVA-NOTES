## **ğŸ”¹ 1. Method Overloading in Java**  

### **ğŸ“Œ Definition**  
Method overloading occurs when **multiple methods in the same class** have the **same name but different parameters** (different number, type, or order of parameters). It allows flexibility in calling methods with different arguments.  

### **ğŸ“ Example: Method Overloading**  
```java

    // Method with two parameters
    int add(int a, int b) {
        return a + b;
    }

    // Overloaded method with three parameters
    int add(int a, int b, int c) {
        return a + b + c;
    }

    // Overloaded method with different parameter types
    double add(double a, double b) {
        return a + b;
    }
}

public class OverloadingExample {
    public static void main(String[] args) {
        MathOperations obj = new MathOperations();
        System.out.println("Sum (2 parameters): " + obj.add(5, 10));       // Calls first method
        System.out.println("Sum (3 parameters): " + obj.add(5, 10, 15));   // Calls second method
        System.out.println("Sum (double parameters): " + obj.add(5.5, 2.5)); // Calls third method
    }
}
```


### **âš  Rules for Method Overloading**  
âœ… Methods **must have the same name**.  
âœ… Methods **must differ in parameters** (number, type, or order).  
âœ… **Return type does NOT matter** for overloading.

## **ğŸ”¹ 2. Method Overriding in Java**  

### **ğŸ“Œ Definition**  
Method overriding occurs when a **subclass provides a new implementation** of a method that is **already defined in its superclass**. The method in the subclass must have **the same name, return type, and parameters** as in the superclass.  

### **ğŸ“ Example: Method Overriding**  
```java
class Animal {
    void makeSound() {
        System.out.println("Animals make sounds.");
    }
}

class Dog extends Animal {
    // Overriding the makeSound() method
    void makeSound() {
        System.out.println("Dog barks.");
    }
}

public class OverridingExample {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.makeSound(); // Calls the overridden method in Dog class
    }
}
```
### **ğŸ”¹ Output:**  
```
Dog barks.  
```

### **âš  Rules for Method Overriding**  
âœ… Methods must have **the same name, return type, and parameters**.  
âœ… Method **must be in a subclass** and override a method from its superclass.  
âœ… **Access modifier cannot be more restrictive** than in the superclass.  
âœ… If the superclass method is `final`, `static`, or `private`, it **cannot be overridden**.

## **ğŸ”¹ 3. Dynamic Method Dispatch (Runtime Polymorphism)**  

### **ğŸ“Œ Definition**  
Dynamic Method Dispatch (also called **Runtime Polymorphism**) occurs when a **reference variable of a superclass** refers to **an object of a subclass**, and the **overridden method in the subclass is called at runtime** based on the actual object type, not the reference type.  

### **ğŸ“ Example: Dynamic Method Dispatch**  
```java

    void show() {
        System.out.println("Parent class method");
    }
}

class Child extends Parent {
    void show() {
        System.out.println("Child class method");
    }
}

public class DynamicDispatchExample {
    public static void main(String[] args) {
        Parent ref; // Superclass reference
        Child obj = new Child(); // Subclass object
        ref = obj; // Reference variable refers to Child object

        ref.show(); // Calls the overridden method in Child class (Runtime Decision)
    }
}
```
### **ğŸ”¹ Output:**  
```
Child class method  
```

### **ğŸ” Explanation**  
Even though `ref` is a `Parent` reference variable, it refers to a `Child` object. **At runtime**, Java determines which method to call **based on the actual object type (`Child`)**, not the reference type (`Parent`) â†’ This is **Dynamic Method Dispatch**.

### **âš  Key Characteristics**  
âœ… Requires **method overriding** in a superclass-subclass hierarchy.  
âœ… Decides method execution **at runtime** based on the actual object type.  
âœ… **Cannot be achieved with static methods** (Static methods are resolved at compile time).
